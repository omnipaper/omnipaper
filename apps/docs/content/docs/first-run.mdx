A fresh install has no baked in config. Register, create a workspace, then connect storage from the app.

## First account

The first person to register becomes the **instance admin**. No seeded credentials: open the app and sign up.

Later accounts are regular users. Only the instance admin reaches instance wide settings (storage, OCR, AI, email security, registration). See [Users, roles & registration](/users-roles).

<Callout>
  Sign up is **closed by default**, but the first account is always allowed so you can bootstrap. Opening registration is a separate step.
</Callout>

## Workspace and checklist

After login you create a **workspace** (required before anything else). An admin checklist then guides three steps:

1. **Connect storage** in Settings → Storage. Required for uploads. See [Storage](/storage).
2. **Upload your first document** once storage is connected.
3. **Set up OCR** in Settings → OCR. Optional but recommended. See [OCR](/ocr).

The checklist updates from real state and disappears when all three are done. Admin only; dismissible per browser.

## Optional later

[OCR](/ocr), [AI](/ai), [email ingest](/email-ingest), and more [users](/users-roles) can wait until you need them.
